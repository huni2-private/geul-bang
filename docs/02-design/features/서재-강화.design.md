| 항목 | Option A (절대 위치) | Option B (전체 높이 Flex) | Option C (고정 Flex) |
|------|---------------------|--------------------------|---------------------|
| 커버 치수 | 64×가변 | 64×카드높이 | **64×80px 고정** |

| 신규 div | 0 | info wrapper | contentWrap |
| Plan 요구사항 충족 | 부분 | 부분 | **완전** |
| 구현 위험도 | 낮음 | 중간 | 낮음 |

## 3. M-1: useNovels.js — loading 상태 추가

### 현재 상태
```js
const [novels, setNovels] = useState([])
const [uploading, setUploading] = useState(false)

useEffect(() => {
  if (!user) return
  const unsubscribe = subscribeNovels(user.uid, setNovels)
  return unsubscribe
}, [user])

return { novels, uploading, uploadNovel, removeNovel }
```

### 변경 후
```js
const [novels, setNovels] = useState([])
const [loading, setLoading] = useState(true)
const [uploading, setUploading] = useState(false)

useEffect(() => {
  if (!user) return
  setLoading(true)
  const unsubscribe = subscribeNovels(user.uid, (data) => {
    setNovels(data)
    setLoading(false)
  })
  return unsubscribe
}, [user])

return { novels, loading, uploading, uploadNovel, removeNovel }
```

**변경 포인트**
- `loading` 초기값: `true` — user가 있을 때 Firestore 첫 스냅샷 전까지 skeleton 표시
- user 변경 시 `setLoading(true)` 재설정 — 계정 전환 후 skeleton 재표시
- subscribeNovels 콜백을 람다로 감싸서 `setLoading(false)` 삽입

## 5. M-3: LibraryPage.jsx — 정렬 드롭다운

### 상수 및 상태

```js
import { useState, useMemo } from 'react'   // useMemo 추가

const SORT_KEY = 'geulbang_sort'
const SORT_OPTIONS = [
  { value: 'lastRead', label: '최근 읽은 순' },
  { value: 'title', label: '제목 순' },
  { value: 'progress', label: '진행률 순' },
]

// LibraryPage 컴포넌트 내부
const [sortOrder, setSortOrder] = useState(() =>
  localStorage.getItem(SORT_KEY) || 'lastRead'
)

function handleSortChange(value) {
  setSortOrder(value)
  localStorage.setItem(SORT_KEY, value)
}

const sortedNovels = useMemo(() => {
  const list = [...novels]
  if (sortOrder === 'title') {
    return list.sort((a, b) => (a.title || '').localeCompare(b.title || '', 'ko'))
  }
  if (sortOrder === 'progress') {
    return list.sort((a, b) => (b.progressRatio || 0) - (a.progressRatio || 0))
  }
  // lastRead: lastReadAt 내림차순, null은 뒤로
  return list.sort((a, b) => {
    const toMs = (ts) => ts?.toMillis ? ts.toMillis() : ts ? new Date(ts).getTime() : 0
    return toMs(b.lastReadAt) - toMs(a.lastReadAt)
  })
}, [novels, sortOrder])
```

```jsx
        <option key={o.value} value={o.value}>{o.label}</option>

```

## 7. Edge Cases

| 케이스 | 처리 |
|--------|------|
| `novel.title`이 빈 문자열 | `charAt(0)` → `''` — 커버에 아무것도 표시 안 됨 (허용) |
| `lastReadAt`이 null (새 소설) | `toMs()` → 0 — lastRead 정렬 시 맨 뒤로 이동 |
| `progressRatio`가 undefined | `|| 0` 처리 — 0%로 취급 |
| 소설 0개이고 loading=false | 온보딩 화면 표시 (정렬 드롭다운 미표시) ✅ |
| user 변경 (익명 → Google) | `setLoading(true)` 재호출 → 스켈레톤 재표시 → 새 novels 로드 |
| localStorage 키 없을 때 | `|| 'lastRead'` fallback ✅ |
