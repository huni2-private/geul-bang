# Geul-bang — 상세 플랜 (Detailed Plan)

> PDCA Phase: **PLAN**
> 문서 버전: v1.0 | 작성일: 2026-04-22 | 상태: Draft

---

## Executive Summary

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Geul-bang (개인용 클라우드 웹 소설 리더) |
| 목표 | .txt 파일 업로드 → 기기 무관 이어읽기 |
| 대상 사용자 | 웹소설 독자 (1인 사용 기준 MVP) |
| 개발 방식 | Firebase BaaS 기반 SPA (React) |
| MVP 목표 기간 | 4주 (Phase 1 기준) |

---

## 1. 문제 정의 (Problem Statement)

| # | 문제 | 근거 |
|---|------|------|
| P-01 | 로컬 파일(.txt)은 기기를 바꾸면 읽던 위치가 사라짐 | 사용자 불편 |
| P-02 | 한국 웹소설 파일은 EUC-KR 인코딩이 많아 글자가 깨짐 | 텍스트 파일 관행 |
| P-03 | Firestore 단일 문서 1MB 제한으로 장편 소설 저장 불가 | DB 제약 |
| P-04 | 별도 앱 설치 없이 즉시 이용 가능한 리더가 없음 | 접근성 문제 |

---

## 2. 솔루션 방향 (Solution Approach)

```
익명 즉시 이용 → txt 업로드 → 인코딩 자동 변환 → 클라우드 저장
→ 기기 무관 이어읽기 → (선택) 계정 연동으로 영구 보존
```

---

## 3. 개발 단계 (Development Phases)

### Phase 1 — MVP (4주)

| 주차 | 목표 | 산출물 |
|------|------|--------|
| 1주 | 프로젝트 초기화, Firebase 연결, 익명 인증 | 인증 플로우 동작 |
| 2주 | 파일 업로드 + 인코딩 변환 + Storage 저장 | 서재 화면 |
| 3주 | 리더 뷰어 + 스크롤 자동 저장 + 이어보기 | 리더 화면 |
| 4주 | 계정 연동, 스마트 헤더, 폰트/테마, QA | 배포 가능 MVP |

### Phase 2 — 기능 확장 (MVP 이후)

| 기능 | 우선순위 |
|------|---------|
| 페이지 모드 (스와이프 방식) | High |
| 이메일 계정 연동 | Medium |
| 소설 내 검색 | Medium |
| 북마크 기능 | Low |
| PWA (오프라인 지원) | Low |
